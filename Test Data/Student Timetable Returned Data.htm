Returned Data

The JSON data that is returned always contains 1 dictionary ("boundaries") and 3 arrays ("clashing", "floating" and "timetable").  The arrays contain timetable items dictionaries.  See the demo page for an example data set.

Boundaries

The boundaries dictionary lists all of the time boundaries that exist in the data set.  This is useful when you are displaying the data in a grid as it lets you know how to set up that grid.  All items in your dataset will start and end at one of the times that are listed.

For example, something like this might be returned:

"boundaries": {
  "830": 1431675000,
  "925": 1431678300,
  "1135": 1431686100,
  "1230": 1431689400,
  "1635": 1431704100
}

This is a list of times in string format to an example timestamp for that time (you can ignore the date part of the timestamp, it is meaningless).  What this particular dictionary is saying is that there is at least one item in your dataset that starts or finishes at 8:30am, at least one that starts or finishes at 9:25am and so on.  There are no items in the dataset that start or finish at a time that isn't listed in the dictionary.

Clashing

The clashing array is a list of timetable items (see below) that clash with an item in the timetable array.  Items in this array don't need to be included in a timetable grid but should be displayed to the user so that they are aware of it.  Clashing items are rare, so this array is usually empty.

Floating

The floating array is a list of timetable items (see below) that don't have a set start and end time.  These are things like drop in workshops or the multigym activity.  It is a lesson that the student is expected to attend once per week.  You can mostly ignore the start and end times for a floating item, although the date part of the timestamp will give you the week that it occurs in.

Timetable Items

A timetable item is a dictionary of key / value fields.  For example:

{
  "Type": "lesson",
  "Id": 38000550273774,
  "Title": "English Literature",
  "Subtitle": "",
  "Start": 1431332700,
  "End": 1431336000,
  "Room": "VY105",
  "Staff": "Graeme Hodgson, Tom Prentki",
  "IsBlank": false,
  "IsCancelled": false,
  "IsRoomChange": false
}

What the fields mean:

Type:

Type describes what the item it is.  Currently this will be one of:

  • "activity" - An extra curricular activity or sports team.
  • "artexambooking" - A booking made by a student for their art exam.
  • "bankholiday" - A national bank holiday.
  • "boardingleave" - Boarding student is on leave from one of the boarding houses.
  • "break" - College break time.
  • "careersappointment" - Appointment with a Careers adviser.
  • "careersweek" - Careers week talk.
  • "exam" - An exam.
  • "holiday" - One of the college holidays.
  • "lesson" - A normal college lesson.
  • "studyperiod" - The spare time between lessons.
  • "studyskills" - A Study Skills appointment.
  • "studysupport" - A Study Support appointment.
  • "trip" - A college trip.
  • "tutorgroup" - Tutor time. 

Id:

The id is normally either going to be a 14 digit integer starting "3800" or a shorter integer.  It will be unique among other items of the same type.

Title and Subtitle: 

This will be some short descriptive text for the item.  Often the subtitle will be blank, but for things like trips you will find that the title is "Trip" and the subtitle is the actual name of the trip.

Start and End:

This is a unix timestamp for when the item starts and ends.

Room:

Which room is the item happening in.  Some items don't have rooms.

Staff:

Which teachers are running the item.

IsBlank:

Is the item not actually a timetabled item?  For instance break time, study periods and college holidays are all considered to be blank.  The value of this field will either be true or false.

IsCancelled:

Has the lesson been cancelled?  If the lesson has been cancelled you should display a message to the user saying so.  The value of this field will either be true or false.

IsRoomChange:

Sometimes lessons need to switch rooms from their normal room.  If this item has switched room you should display an appropriate message to make the user aware.  The value of this field will either be true or false.
